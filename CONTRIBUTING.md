# Contributing to Athos

Contributions are welcome. Please keep changes focused, documented, and compatible with the existing structure of the project.

## Guidelines

- Prefer small, reviewable changes.
- Keep documentation and skills aligned with the actual repository state.
- Avoid committing secrets, credentials, or private data.
- If you are changing behavior relevant to installation or memory handling, update the related documentation.

## Suggested process

1. Open an issue describing the change.
2. Create a branch with a clear name.
3. Make the change and validate the affected files.
4. Submit a pull request with context and testing notes.
